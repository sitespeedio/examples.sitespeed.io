Rules definition

The rules are simple: Green color means you don't need to do anything (of course there could still be things you can do to get better performance). Yellow tells you that you should look into this, and guess what, red means that something is really bad. The blue color are things for your information, no rules but things that will be good to know.

  • Sitespeed score are based on the rules (modified and enhanced Yslow rules): Over 90 is good, between 80-90 is not so good, and below 80 is bad, really bad.
  • The number of requests: Below 25 is good, between 25-88 is something you should look into (88 is the average amount of requests a site has, collected from http://httparchive.org) and above 88 is really bad.
  • The weight of a page: Below 1 mb is ok (but really too large for mobile), between 1-1.2 mb is something you should investigate & heavier is bad. 1.2 mb is the average site weight, statistics from http://httparchive.org.
  • The document weight: Below 10 kb is good (gzipped), between 10-15 is ok (but can be better) and above is bad. A small document is good for mobile.
  • The number of javascripts: Two or less is good, up to 15 means you should look into it and more than 4 means you need to fix it. 15 is the average amount js request, statistics from http://httparchive.org.
  • The number of stylesheets: One is good, more than five is really bad. Five is the average amount of CSS files, data from http://httparchive.org.
  • The number of stylesheet images: Two is good, above four is bad.
  • The number of images: 25 or less images is OK, below 54 (the average from http://httparchive.org) is ok, and above is bad.
  • The number of requests for a primed cache: 2 requests is OK else if not 90% of the request are cached, it is bad.
  • The number of pages with the possibility of frontend single point of failure: 0 pages is good, everything else is bad. Excludes subdomains on the document domain from the spof.
  • Average number of javascripts loaded synchronously inside of the head tag: 0 is good, 1 or more is bad and not OK.
  • Average number of requests for static content without a far-future expiration date: 0 is good. More than 4 is bad. Google Analytics & Gaug.es is excluded from this.
  • The critical path (every file loaded inside of head, will postpone the rendering of the page, try to avoid loading javascript synchronously, load files from the same domain as the main document, and inline css for really fast critical path): You should always aim for score 100, above 95 is OK.
